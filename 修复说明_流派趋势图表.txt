大众金曲网站 流派趋势图表修复说明


问题描述
────────────────────────────────────────────────────────────────────

网页中的"流派趋势"模块的图表（折线图、雷达图、热力图、散点图、柱状图、面积图、漏斗图）无法正常显示。


问题根源分析
────────────────────────────────────────────────────────────────────

位置：app.py 文件中的 get_trends() 函数（第141-157行）

原问题代码：
    def get_trends():
        """获取流派趋势"""
        ensure_data()
        trends = defaultdict(lambda: defaultdict(int))
        for entry in DATA:
            year = int(entry['year'])
            genre = entry['genre']
            trends[genre][year] += 1

        main_genres = ['pop', 'rock', 'hip-hop', 'electronic', 'r&b']
        result = {}
        for genre in main_genres:
            result[genre] = {
                'label': genre.upper(),
                'data': [trends[genre].get(y, 0) for y in range(1995, 2021)]
            }
        return result

问题分析：
────────────────────────────────────────────────────────────────────

1. 原函数返回的是每年的"歌曲数量"（1 for each song）
   • trends[genre][year] += 1 统计的是歌曲数，不是热度

2. 前端期望的是"平均热度值"（0-100的分数）
   • ECharts 图表需要在 0-100 范围内的数值
   • 用户想看的是热度趋势，而不是数量趋势

3. 导致的结果：
   • 后端返回: [1, 1, 1, 1, 1, ...] （歌曲数量）
   • 前端期望: [85, 87, 89, 90, 91, ...] （平均热度）
   • 图表无法正常绘制


修复方案
────────────────────────────────────────────────────────────────────

替换 get_trends() 函数为正确的实现：

def get_trends():
    """获取流派趋势"""
    ensure_data()

    main_genres = ['pop', 'rock', 'hip-hop', 'electronic', 'r&b']
    result = {}

    for genre in main_genres:
        yearly_data = defaultdict(lambda: {'count': 0, 'popularity': 0})
        for entry in DATA:
            year = int(entry['year'])
            if entry['genre'] == genre:
                yearly_data[year]['count'] += 1
                yearly_data[year]['popularity'] += int(entry['popularity'])

        # 计算每年的平均热度
        data = []
        for year in range(1995, 2021):
            if year in yearly_data:
                avg_popularity = yearly_data[year]['popularity'] / max(yearly_data[year]['count'], 1)
                data.append(int(avg_popularity))
            else:
                data.append(0)

        result[genre] = {
            'label': genre.upper(),
            'data': data
        }

    return result


关键改进：
────────────────────────────────────────────────────────────────────

1. 为每个流派和年份统计：
   • count: 该流派该年的歌曲数量
   • popularity: 该流派该年所有歌曲的热度总和

2. 计算平均热度：
   • 平均热度 = 总热度 / 歌曲数量
   • 取整数值（0-100）

3. 为所有26年填充数据：
   • 确保数据完整性（1995-2020）
   • 没有数据的年份使用 0 填充

前端图表展示
────────────────────────────────────────────────────────────────────

修复后，前端 ECharts 可以正确渲染以下图表：

1. 折线图（Line Chart）
   • 显示5条折线，分别代表5个流派
   • X轴：年份（1995-2020）
   • Y轴：平均热度（0-100）
   • 特征：可见流派的升降趋势

2. 柱状图（Bar Chart）
   • 显示分组柱状图
   • 易于对比不同流派的热度差异

3. 面积图（Area Chart）
   • 显示流派热度的堆叠面积
   • 直观展示总体热度的变化

4. 散点图（Scatter）
   • 显示热度分布的离散点
   • 便于发现异常值

5. 雷达图（Radar）
   • 多维对比5个流派的特征
   • 每条轴代表一个年份

6. 热力图（Heatmap）
   • 二维矩阵：行为流派，列为年份
   • 颜色深浅表示热度高低

7. 漏斗图（Funnel）
   • 显示流派平均热度的排序

常见问题排查
────────────────────────────────────────────────────────────────────

问题：修改后图表仍不显示
排查步骤：
  1. 确认 Web 服务器已重启
  2. 清空浏览器缓存（Ctrl+Shift+Delete）
  3. 刷新页面（F5）
  4. 打开开发者工具检查 Network 和 Console

问题：某个流派的数据显示为空
排查步骤：
  1. 检查该流派是否有数据：
     grep "流派名称" processed_data.csv | wc -l
  2. 确保流派名称拼写正确（小写，使用 '-' 连接）
  3. 验证 CSV 文件格式是否正确

问题：热度值过大或过小
排查步骤：
  1. 检查计算公式是否正确
  2. 验证 CSV 数据中的 popularity 字段范围
  3. 确认整数转换是否丢失精度
